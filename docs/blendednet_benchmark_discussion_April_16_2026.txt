BLENDEDNET BENCHMARK DISCUSSION
Started: April 16, 2026
================================================================================

OVERVIEW
--------
BlendedNet is a Blended Wing Body (BWB) aircraft aerodynamic optimization
environment. It uses a Transolver surrogate (same architecture as DrivAer_Star)
to predict aerodynamic fields, with the shapebench_5 multi-point drag
minimization reward.

This file documents the benchmark runs of PSO (GA framework), L-BFGS-B, and
v3_dynamic_optimizer on BlendedNet / shapebench_5.


ENVIRONMENT
-----------
Environment:  BlendedNet
Reward:       shapebench_5
Surrogate:    Transolver (pre-trained, frozen inference)

Design space: 10 continuous parameters
  B1:    [100.0, 200.0]   — fuselage width param 1
  B2:    [50.0,  200.0]   — fuselage width param 2
  B3:    [200.0, 700.0]   — fuselage width param 3
  C2:    [550.0, 850.0]   — wing chord param 2
  C3:    [180.0, 280.0]   — wing chord param 3
  C4:    [60.0,   90.0]   — wing chord param 4
  S1:    [40.0,   60.0]   — surface param 1
  S2:    [40.0,   60.0]   — surface param 2
  S3:    [24.0,   40.0]   — surface param 3
  alpha: [-3.0,    3.0]   — angle of attack (degrees)

No discrete variables.

Flight conditions: Mach=0.3, Re=1e7 (fixed for all runs)


REWARD: shapebench_5
--------------------
Multi-point drag minimization across 5 CL operating points.
  CL targets: [0.206, 0.206, 0.206, 0.185, 0.227]
  (3 unique CL targets after deduplication)

For each unique CL target, bisection search (8 iterations) finds the angle of
attack alpha that achieves that CL. This makes each design evaluation invoke
3 × 8 = 24 Transolver forward passes.

Reward: R = -mean_CD  (mean drag coefficient across the 5 operating points)
Fail reward: -5.0

Approximate cost per design evaluation: ~5-10s (24 surrogate calls at ~0.1-0.5s
each, including bisection overhead).

DESIGN OF EXPERIMENTS NOTES
-----------------------------
- L-BFGS-B has the smallest budget (~600 evals) because it uses finite-difference
  gradients: each gradient step internally costs O(dim) = O(10) evaluations.
  The optimizer is informed per step, so fewer raw evals are needed compared to
  population-based methods. Expect fast initial convergence but sensitivity to
  local optima.

- PSO uses 4000 evals, the largest budget. As a population-based global search
  method with no gradient information, it needs more evaluations to explore the
  10D space. The prior 2000-eval run may have been too short to fully converge.

- v3 uses 2000 evals (200 iter × 10 per iter). The LLM oracle provides
  high-level reasoning about which regions to explore, which can offset the
  smaller raw eval budget. Each iteration: 1 LLM-proposed s0 design + 9
  sequential oracle-guided designs.

- All 3 frameworks are compatible with BlendedNet (no discrete variables,
  get_param_bounds() and write_design() implemented).

- DrivAer_Star BO_torch runs are NOT being run here — BO_torch is already
  running 20 seeds each for vtk_F and vtk_N on DrivAer_Star, and BlendedNet
  does not have a vtk-based parameterization (uses FFD mesh deformation via VSP).
  BO_torch for BlendedNet could be added in a future benchmark pass.
